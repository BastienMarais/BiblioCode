Structure de base et informations utiles :

Code :

<!doctype html>
<html lang="fr">
	<head>
		<meta charset="utf-8">
		<title> Nom_Onglet </title>
	</head>
	<body>
		Content...
	</body>
</html>

Description :

Tout document HTML doit contenir cette structure afin de respecter les normes W3C. Pour savoir si votre code est en adéquation avec les normes W3C vous pouvez aller sur ce site. Un autre site utile est jsbeautifier, il permet d'uniformiser votre mise en forme sans faire d'erreurs.

Les balises de regroupement :

Code :

<div> ou <header> ou <footer>
	Content...
</div> ou </header> ou </footer>

ou <article> ou <content>
	Content...
ou </article> ou </content>

<p>
	texte_normal
	<span>
		texte_modifié
	</span>
</p>

Description :

Les balises <header>, <footer>, <content> et <article> sont des balises équivalentes a une balise <div>, elles ont été ajouté dans le HTML 5 pour être un peu plus précise dans leur contenu mais ça reste des <div>. Ces balises permettent de regrouper votre code en compartiment c'est pratique pour associer un style a une partie de votre code. Par exemple sur ce site chaque <article> a un fond gris foncé.

Les balises <span> quand a elles servent souvent a modifier quelques parties d'un paragraphe quand elles sont associées a un style css.

Les liens :

Code :

<a href="cible_du_lien">
	Texte surligné et bleu qui sera le lien
</a>

<a href="cible_du_lien" target="blank">
	Texte...
</a>

<a id="id_ancre"> </a>
<a href="#id_ancre"> Aller vers id_ancre </a>

Description :

Pour utiliser un lien il faut passer par la balise <a>, on indique ensuite vers quoi pointe ce lien (un id sur la page,une autre page html ou un http://... ) avec href.
Vous pouvez ajouter target="blank" dans la balise afin que ce lien s'ouvre dans un autre onglet plutot que de remplacer la page actuelle (option par défaut).



Les images et favicon :

Code :

<img src="../images/mon_image.png"
	width="largeur_dimension"
	height="hauteur_dimension"
	alt="description_photo_malvoyant" />

<head>
	<link rel="shortcut icon"
		href="../images/favicon.ico"
		type="image/x-icon">
</head>

Description :

Pour insérer une image il suffit d'indiquer dans la balise <img> le chemin d'accès jusqu'a cette image (src), des dimensions si vous le souhaitez (height et width) et pour les normes W3C une description (alt).

Ensuite un favicon est l'image qu'on peut apercevoir a coté du nom de l'onglet. Il faut absolument que ce soit un .ico, vous pouvez utiliser ce site ou celui-ci pour convertir vos images en .ico .



La balise video :

Code :

<video controls autoplay loop >
	<source src="../vidéo/video.webm" type="video/webm">
	<source src="../vidéo/video.mp4" type="video/mp4">
	<source src="../vidéo/video.ogv" type="video/ogg">
	Votre navigateur ne permet pas de lire les vidéos HTML5.
</video>

Description :

Cette balise vous permet d'intégrer des vidéos dans votre page HTML. La balise <video> peut contenir l'attribut "controls" ajoutant les boutons de base pour une vidéo, l'attribut "autoplay" pour déclencher le lancement automatique de la vidéo et l'attribut "width" pour définir la taille. Il y a également "loop" pour une lecture en boucle.

La balise <source> quand a elle contient le chemin d'accès vers la vidéo et son type. Mettre plusieurs <source> permet en cas de problème de format d'en avoir un compris par le navigateur.



La balise audio :

Code :

<audio controls autoplay loop>
	<source src="../music/music.ogg" type="audio/ogg" >
	<source src="../music/music.mp3" type="audio/mpeg" >
</audio>

Description :

La balise <audio> permet d'ajouter du son a votre page ou juste un document audio. La balise peut contenir l'attribut "controls" pour donner le choix a l'utilisateur de l'écouter ou non et aussi l'attribut "autoplay" pour un lancement automatique. Il y a également "loop" pour une lecture en boucle. Par exemple avec seulement autoplay et loop on obtient une page avec une musique d'ambiance.

La balise <source> quand a elle contient le chemin d'accès vers le document audio et son type. Mettre plusieurs <source> permet en cas de problème de format d'en avoir un compris par le navigateur.



Fieldset et legend

Code :

Créer un encadrer :
<fieldset> Contenu du cadre </fieldset>

Ecrire sur l'encadrer :
<fieldset>
	<legend> Texte a afficher sur le cadre  </legend>
	Contenu du cadre
</fieldset>

Description :

La balise <fieldset> permet d'ajouter un cadre autour de tout ce qui est entre lui et sa balise fermante.

La balise <legend> quand a elle permet d'ajouter du texte dans le coin haut droit du cadre.



Utilisation de la balise map :

Code :

<img src="../images/map_global.png"
usemap="#map" alt="..."/>

<map name="map">
	<area shape="circle"
	coords="x,y,rayon"
	href="cible_lien_1"
	alt="..."/>

	<area shape="rect"
	coords="x1,y1,x2,y2"
	href="cible_lien_2"
	alt="..."/>

	<area shape="poly"
	coords="x1,y1,x2,y2,..."
	href="cible_lien_3"
	alt="..."/>
</map>

Description :

Une map est une image a laquelle on définit plusieurs zones. Par exemple quand on clique sur une zone particulière de l'image on est rediriger vers un une autre page etc...

Pour l'utiliser il faut rajouter usemap="name_map" dans la balise <img>. Ensuite on ouvre une balise <map> avec comme name ce qu'il y a dans votre usemap.

Maintenant il faut définir les zones cliquables et pour cela vous avez le choix entre des rectangles, des cercles et des polygones. Pour les définir c'est une balise <area> avec un attribut shape="rect/circle/poly" et un attribut coords="liste des points pour définir la forme choisie".



Balise code et coloration syntaxique

Code :

<head>
	<link rel="stylesheet" href="https://cdnjs.cloudflare.com/ajax/libs/highlight.js/9.12.0/styles/monokai-sublime.min.css">
	<script src="https://cdnjs.cloudflare.com/ajax/libs/highlight.js/9.12.0/highlight.min.js"></script>

</head>
<body>
	<pre><code>
		Code non interprété...
	</code></pre>
</body>

Description :

Pour ajouter la coloration syntaxique a vos codes non interprétés vous devez mettre ces lignes la dans la balise <head>

Pour votre insérer du code il vous suffit de mettre les balises <pre><code>.



Les formulaires :

Code :

Base d'un formulaire :
<form action="cible_formulaire" method="POST ou GET">
	Insérez ici les éléments vu ci-dessous...
	<input type="submit" value="texte afficher dans le bouton"/>
</form>

Champs de texte :
<label for="nom_lie_label"> Texte du label </label>
<input id="nom_lie_label" type="text" required="required" name="nom_variable"/>

Zone de texte :
<label for="id_zone" > Texte du label </label>
<textarea id="id_zone" name="Commentaire"> Texte déjà inscrit dans la zone </textarea>

Cases a cocher :
<input type="checkbox" name="nom_variable[]" value="valeur 1" /> texte a afficher a coté de la case 1 <br/>
<input type="checkbox" name="nom_variable[]" value="valeur 2" /> texte a afficher a coté de la case 2 <br/>
<input type="checkbox" name="nom_variable[]" value="valeur 3" /> texte a afficher a coté de la case 3

Menu défilant :
<select>
	<option name="nom_variable" value="valeur 1"> texte a afficher dans le menu 1 </option>
	<option name="nom_variable" value="valeur 2"> texte a afficher dans le menu 2 </option>
	<option name="nom_variable" value="valeur 3"> texte a afficher dans le menu 3 </option>
	<option name="nom_variable" value="valeur 4"> texte a afficher dans le menu 4 </option>
</select>

Description :

Pour commencer un formulaire il faut une balise <form> avec comme attributs possibles "action" qui peut être un script a exécuter en cas d'envoi de formulaire et "method" qui a pour valeur soit "GET" soit "POST" (GET = éléments du formulaire visible dans l'url tandis que POST non).
Ensuite ce formulaire peut contenir des champs de texte, des zones de texte, des checkbox ou des boutons radio ou encore des menus défilants... C'est comme vous voulez !
Le plus important c'est que le formulaire finisse par un <input> de type="submit" car c'est lui qui envoi le formulaire.

La balise <label> permet que lorsqu'on clique sur le texte du champs, le champs soit pré-sélectionné sans qu'on est réellement besoin de cliquer dans le rectangle du champs. Il faut lui indiquer dans l'attribut "for" l'id du champs auquel il est lié.

Vous pouvez rendre avec des champs obligatoire avec l'attribut "required" et vous pouvez aussi pré-sélectionner des boutons radio avec l'attribut "checked".
Il existe d'autres types de bouton (comme le type="color") mais ceci sont les principaux.
